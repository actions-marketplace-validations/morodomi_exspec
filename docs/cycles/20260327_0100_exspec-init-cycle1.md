## Implementation Notes

### Goal
`cargo install exspec` 後の導入摩擦を削減する。プロジェクトのフレームワークを自動検出し、最適な `.exspec.toml` を生成する。v0.6.0 のメイン機能。

### Background
現状、ユーザーは `.exspec.toml` を手動で作成する必要がある。`exspec init` を実行するだけで言語を自動検出し、適切な設定ファイルを生成できるようにする。

### Design Approach
非対話式。ファイルシステム走査のみで検出する。CLI は `clap` の `Subcommand` として `Init(InitArgs)` を追加し、`crates/cli/src/main.rs` 内に `run_init()` / `detect_languages()` / `generate_toml()` を実装する。`crates/cli/src/init.rs` を新設し、detect_languages / generate_toml / run_init をそちらに置く。main.rs は run_init の呼び出しのみ。detect_languages は純粋関数 (Vec<String> を返すのみ)。警告メッセージは run_init が担う。

検出優先度順 (複数言語は全て列挙):

| 言語 | トリガーファイル |
|------|----------------|
| Python | `pyproject.toml`, `requirements.txt`, `setup.py` のいずれか |
| TypeScript | `package.json` |
| PHP | `composer.json` |
| Rust | `Cargo.toml` |

生成される最小 `.exspec.toml`:

```toml
# Generated by `exspec init`

[general]
lang = ["python"]

[paths]
ignore = [".venv", "__pycache__"]
```
