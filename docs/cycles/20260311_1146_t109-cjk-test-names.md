## Implementation Notes

### Goal
CJK (Japanese/Chinese/Korean) テスト名がT109で誤検出されるFPを修正する。

### Background
ShaRecoドッグフーディングで発見。CJK言語はスペースやアンダースコアを単語区切りに使わないため、`is_single_word` heuristicが誤検出する。Root causeは `rules.rs:is_undescriptive_test_name()` の2箇所: no-prefix branchのsingle-word判定とsuffix branchのbyte-length判定。

### Design Approach
非ASCII文字を含む名前はsingle-word heuristicをスキップする。`normalized.chars().any(|c| !c.is_ascii())` でCJK含有を検出し、trueならsingle-word/short-wordチェックをバイパス。GENERIC_TEST_NAMESブラックリストチェックは維持(CJK名がブラックリストに一致することはない)。
